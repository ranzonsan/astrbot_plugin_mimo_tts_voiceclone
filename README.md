## 功能特性

- 支持用户上传参考音频进行声音克隆
- 支持为多个不同的 aiocqhttp 机器人 ID，配置专属参考声音
- 可配置触发声音克隆的概率
- 支持多种音频格式输出（WAV、MP3、OGG、FLAC）
- 可调节语音播放速度
- 仅对 LLM 生成的回复进行处理，其他插件传入的回复不会触发 TTS

## 配置说明

在 AstrBot WebUI 的插件配置页面中，您可以配置以下参数：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| API Key | Mimo TTS 服务的 API 密钥 | - |
| API Base URL | Mimo TTS API 的基础 URL | https://api.xiaomi.com/v1 |
| 触发概率 | LLM 回复触发声音克隆的概率 (0-1) | 0.5 |
| 默认参考音频 | 未匹配到机器人 ID 或对应声音时使用的参考音频文件 | - |
| 机器人专属参考声音文件池 | 上传所有机器人专属参考声音文件 | [] |
| 按机器人 ID 指定参考声音 | 可新增多条机器人 ID 与文件名/路径的对应关系，仅对 aiocqhttp 生效 | [] |
| 模型名称 | 使用的 TTS 模型名称 | mimo-v2.5-tts-voiceclone |
| 输出格式 | 生成的音频文件格式 | wav |
| 语音速度 | 语音播放速度 (0.5-2.0) | 1.0 |
| 最大文本长度 | 单次 TTS 转换的最大文本长度 | 500 |

## 使用方法

1. 在 AstrBot WebUI 中安装此插件
2. 配置 Mimo TTS 的 API Key 和 Base URL
3. 上传默认参考音频文件：
   - 支持 WAV、MP3、FLAC、OGG 格式
   - 时长需要在 5 - 30s，无背景音干扰
4. [可选配置] 如果需要让多个机器人使用不同的声音，则进行以下操作：
   - 首先，在“机器人专属参考声音文件池”中，上传**所有**专属参考音频；
   - 在“按机器人 ID 指定参考声音”中，点击“添加条目”；
   - “机器人的 ID”，填写 AstrBot 消息对象中的 `self_id`，OneBot v11 中，ID 通常为机器人的 QQ 号
   - “该机器人调用的参考声音文件名” 中，填写文件池中已上传文件的文件名，例如 `123456.wav`
5. 调整触发概率和其他参数
6. 保存配置并启用插件


## 工作原理

1. 插件监听 LLM 生成的文字回复
2. 根据配置的概率决定是否触发声音克隆
3. 对 aiocqhttp 协议消息，通过 `event.get_self_id()` 获取机器人 ID
4. 如果配置了该机器人 ID 的专属参考声音，使用专属声音调用 MiMo-v2.5-TTS-VoiceClone TTS API
5. 如果未获取到机器人 ID 或未配置对应声音，回落到默认参考音频
6. 用生成的语音替换原始文字回复。如果 TTS 失败，保留原始文字回复

## 依赖要求

- Python >= 3.10
- aiohttp >= 3.8.0
- AstrBot >= 4.0.0
